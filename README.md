# K-Means Performance Benchmark: CPU vs. GPU

Performance analysis of the K-Means clustering algorithm across different hardware architectures. Demonstrates parallel processing advantages for ML workloads by comparing sequential CPU implementations against GPU-accelerated variants on both NVIDIA and AMD hardware.

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| Languages | Python 3.x, C++17, CUDA C++ |
| CPU | scikit-learn, NumPy, pandas, matplotlib |
| NVIDIA GPU | RAPIDS cuML, CUDA Toolkit 11.x |
| AMD iGPU | Taichi ≥1.7 — Vulkan backend (`pip install taichi`, no ROCm needed) |
| Cloud | Google Colab — free NVIDIA T4 GPU |

## ⚙️ HPC Design Decisions

| Decision | Rationale |
|---|---|
| `float32` everywhere | GPUs are optimized for FP32; `double` would skew bandwidth comparisons |
| Flat 1D row-major array in C++ | No heap fragmentation — identical memory layout to GPU VRAM |
| Squared Euclidean distance (no `sqrt`) | Monotonic — doesn't change nearest-centroid result, saves cycles |
| Isolated timing | Timer wraps strictly the Lloyd loop — I/O and data generation excluded |
| `n_init=1` in scikit-learn | Default of 10 restarts would be unfair vs cuML and C++ which run once |
| `algorithm="lloyd"` in scikit-learn | Forces standard Lloyd — identical algorithm to all GPU variants |
| K-Means++ with `random_state=42` | Deterministic initialization across all variants for reproducibility |
| JIT warmup before timing (Taichi) | First kernel call compiles to SPIR-V — excluded from measured time |

## ▶️ Quick Start

### Sequential — local
```bash
git clone https://github.com/andreisugu/KMeans-GPU-Benchmark.git
cd KMeans-GPU-Benchmark
pip install -r requirements.txt

# Python sklearn
cd src/baseline && python benchmark_cpu.py

# C++ sequential
make && ./kmeans_seq --all
```

### Parallel #1a — RAPIDS cuML (Google Colab)
1. Open `notebooks/KMeans_Benchmark.ipynb`
2. `Runtime > Change runtime type > T4 GPU`
3. Run all cells

### Parallel #1b — Taichi iGPU (local, Radeon 780M)
```bash
pip install taichi
python -c "import taichi as ti; ti.init(arch=ti.vulkan); print('OK')"
cd src/taichi && python benchmark_taichi.py
```

### Visualize
```bash
cd results && python plot_results.py
```
